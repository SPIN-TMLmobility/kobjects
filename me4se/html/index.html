ME4SE

The purpose of this project is to make Java 2 Micro Edition APIs like LCDUI and the Generic Connection Framework available for the Java 2 Standard Edition.

Motivation:

  1. Provide some limited development support for platforms where no emulator is available.
  2. Allow demonstration of MIDlets before installantion on the device.
  3. Perhaps allow running MIDlets with Personal Java (not tested yet)

Examples

  • Karl Hörnell's MidpMan
  • DS Effects DS_Fire (Other DS effects work fine, too. Unfortunatley, a shared class name seems to confuse browsers when I put more than one on this page)
  • Spruce MIDlets:
    • Spruce Blocks
    • Spruce Stopwatch
    • Spruce Tester
Please Note
  • The Applets may not work if a CLASSPATH is set on your system. In that case, please use ienocp.bat or nsnocp.bat to start your browser. You may need to adopt the files to your system environment.
  • The sixlegs.com png library is used for PNG support.
I have also tested the SpaceInvaders and Snake examples contained in the Java 2 Micro Edition Wireless Toolkit. If I get the permission from SUN, I will add them to this page.

Instructions

The following command starts a MIDlet on your desktop computer if me4se.zip and the sixlegs png library are contained your classpath: 

   java org.kobjects.me4se.MIDletRunner YourMIDletName

Since the emulation is not complete yet, your MIDlet may not work. If so, please report your problem to the address given below. Alternatively, you can fix the emulation and contribute your additions to the project.
